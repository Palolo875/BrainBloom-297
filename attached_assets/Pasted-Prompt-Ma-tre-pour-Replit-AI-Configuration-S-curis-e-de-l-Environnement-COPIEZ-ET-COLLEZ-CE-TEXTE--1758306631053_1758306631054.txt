Prompt Maître pour Replit AI : Configuration Sécurisée de l'Environnement[COPIEZ ET COLLEZ CE TEXTE DANS LE CHAT DE REPLIT AI]Objectif : Configurer le projet pour gérer les clés d'API de manière sécurisée et portable en utilisant un fichier d'environnement .env.local.Règles Strictes :1.N'utilise PAS l'outil "Secrets" de Replit.2.N'écris JAMAIS de vraies clés API dans aucun fichier.3.Suis les instructions à la lettre.Instructions Pas-à-Pas :Étape 1 : Créer le fichier d'environnement modèle.•À la racine du projet, crée un fichier nommé .env.example.•Ce fichier sert de modèle pour savoir quelles clés sont nécessaires.•Remplis .env.example avec le contenu exact suivant :# ==================================================
#       VARIABLES D'ENVIRONNEMENT POUR BRAINBLOOM
# ==================================================
#
# Instructions :
# 1. Renommez ce fichier en ".env.local"
# 2. Remplacez les valeurs ci-dessous par vos vraies clés.
# 3. NE JAMAIS PARTAGER LE FICHIER .env.local

# --- Clés publiques pour le client (navigateur) ---
NEXT_PUBLIC_SUPABASE_URL="VOTRE_URL_SUPABASE_ICI"
NEXT_PUBLIC_SUPABASE_ANON_KEY="VOTRE_CLE_ANON_SUPABASE_ICI"

# --- Clés secrètes pour le serveur ---
# Clé secrète pour les opérations d'administration Supabase (si nécessaire)
SUPABASE_SERVICE_KEY="VOTRE_CLE_SERVICE_ROLE_SUPABASE_ICI"

# Clé secrète pour l'API d'inférence Hugging Face
HF_TOKEN="VOTRE_TOKEN_HUGGING_FACE_ICI"
Étape 2 : Sécuriser le fichier des secrets.•Ouvre le fichier .gitignore.•Vérifie que la ligne .env.local est bien présente. Si elle n'y est pas, ajoute-la à la fin du fichier. C'est l'étape de sécurité la plus importante.Étape 3 : Confirmer la fin de la mission.•Une fois les deux étapes terminées, écris simplement : "Configuration de l'environnement terminée. Le fichier .env.example a été créé et .gitignore a été vérifié. L'utilisateur peut maintenant créer son fichier .env.local et y insérer ses clés."Votre Marche à Suivre1.Donnez ce prompt à Replit AI. Il va exécuter ces 3 étapes en quelques secondes.2.Une fois que l'IA a confirmé avoir terminé, vous verrez un nouveau fichier .env.example dans votre projet.3.Faites un clic droit sur .env.example -> "Rename" -> et nommez-le .env.local.4.Ouvrez ce nouveau fichier .env.local et collez vos vraies clés à la place des "VOTRE_..._ICI".Et voilà. Votre projet est maintenant configuré de manière professionnelle. Vous n'aurez plus jamais à utiliser l'interface des "Secrets" de Replit pour ce projet. Si vous le déplacez, il vous suffira de recréer ce simple fichier texte.
